Python Project
Report Automation System
AUTOMATED REPORT SCHEDULING & DELIVERY ENGINE
A production-grade Python system that automates the full lifecycle of business reports: reading configuration from SharePoint Excel files, running SQL readiness checks across multiple databases (MSSQL, Greenplum, StarRocks), refreshing Excel workbooks via COM automation, and delivering them by email through Outlook with a formatted data table pasted inline. A real-time Slack dashboard renders per-report delivery status as a PNG image. Built and deployed at Space International — the fintech behind TBC Bank Uzbekistan.
THE PROBLEM I SOLVED
Before
Every day, someone had to manually open each Excel report, wait for it to refresh from the database, copy the table, paste it into an Outlook email, and send it to the right people — repeated for every report, every day.
After
A Python system runs unattended on a server — checking data readiness, refreshing Excel, composing and sending emails automatically at scheduled times. The team manages everything by editing a single Excel config file. No code changes needed to add or reschedule a report.

MODULE DOCUMENTATION
SharePoint Client
sharepoint_client.py
File Access Layer
Provides a unified interface for reading files from either a local OneDrive-synced folder (testing) or Microsoft SharePoint Online via the Graph API (production). A factory function picks the right implementation from a JSON config flag — all downstream code stays identical in both modes.
  • OAuth2 token acquisition via MSAL with automatic refresh on 401 responses
  • Methods: download_bytes, download_text, download_to_file, download_excel
  • LocalClient maps SharePoint-style paths to local filesystem paths for zero-config testing
  • Lazy site ID and drive ID resolution with caching to minimise redundant API calls
sharepoint_client.py — factory pattern
def build_client(config: dict):
    if config.get("local_mode", False):
        return LocalClient.from_config(config)      # reads from synced OneDrive folder
    else:
        return SharePointClient.from_config(config) # Microsoft Graph API

class SharePointClient:
    def _acquire_token(self) -> str:
        app = msal.ConfidentialClientApplication(
            self.client_id,
            authority=f"https://login.microsoftonline.com/{self.tenant_id}",
            client_credential=self.client_secret,
        )
        result = app.acquire_token_for_client(
            scopes=["https://graph.microsoft.com/.default"]
        )
        return result["access_token"]
Config Loader
config_loader.py
Configuration & Scheduling Logic
Reads a master Excel workbook with three sheets — Reports, SQLChecks, Servers — and hydrates typed dataclasses. Thread-safe with a 60-second TTL cache so the live scheduler can hot-reload config without restarting.
  • Flexible send_days parser supports: *, Mon-Fri, Mon,Wed,Fri, and single-day formats
  • should_send_today() called by the scheduler before each job to skip off-schedule days
  • SQL file encoding auto-detection: tries utf-8-sig → utf-8 → cp1252 → latin-1 → utf-16
  • Supports three server types: MSSQL, Greenplum, StarRocks
config_loader.py — send_days parser
def _parse_send_days(raw: str) -> tuple[bool, dict]:
    if not val or val == "*":
        return True, {d: True for d in WEEKDAYS}       # every day

    if "-" in val and "," not in val:                   # range: Mon-Fri
        start  = WEEKDAYS.index(parts[0].capitalize()[:3])
        end    = WEEKDAYS.index(parts[1].capitalize()[:3])
        active = {d: (start <= i <= end) for i, d in enumerate(WEEKDAYS)}
        return False, active

    if "," in val:                                      # list: Mon,Wed,Fri
        days_on = {d.strip().capitalize()[:3] for d in val.split(",")}
        return False, {d: (d in days_on) for d in WEEKDAYS}
Main Orchestrator
main.py
Core Engine
The heart of the system. Handles SQL readiness checks across multiple databases, Excel COM automation (refresh + visible range detection + clipboard copy), Outlook COM email composition with an inline formatted table, and a retry loop with hard deadline enforcement.
  • Excel refresh via win32com with a watchdog timer that force-kills Excel if it hangs past the timeout
  • detect_visible_used_range() skips hidden rows/columns and handles merged cells correctly
  • Runtime credential patching for OLEDB/ODBC connections — no passwords stored in Excel
  • Retry every 15 minutes until 23:45, then marks as FAILED. Cancels if report is deactivated mid-retry
  • 30 concurrent jobs via APScheduler ThreadPoolExecutor with coalesce=True

Sent Log
sent_log.py
Delivery Tracking
Thread-safe JSON persistence layer tracking delivery status of every report. On startup, auto-cleans entries older than 10 days. Used by the scheduler to prevent duplicate sends and by Slack notifier to render per-report status.
  • Shared threading.Lock prevents write conflicts across 30 concurrent scheduler threads
  • Each entry stores: time, status (OK / LATE / FAILED), date
  • Auto-pruning on load removes entries older than 10 days to keep the file compact
sent_log.py — thread-safe write
def update_sent_log(report_id: str, status: str):
    with _sent_log_lock:          # blocks all other threads while writing
        sent_log[report_id] = {
            "time":   datetime.now().strftime("%H:%M"),
            "status": status,     # "OK" | "LATE" | "FAILED"
            "date":   datetime.now().strftime("%Y-%m-%d"),
        }
        save_sent_log()           # persist to disk immediately

Auto Refresh
auto_refresh.py
Config Hot-Reload
Runs as a background daemon thread, reloading report configuration every 5 minutes and syncing changes to the live scheduler without any restart. Detects and logs additions, removals, reschedules, and activate/deactivate events to the Slack log channel.
  • Snapshots config before and after refresh to diff changes with precision
  • Calls sync_scheduler() to add or reschedule only changed jobs
  • Posts change summary to Slack log channel and refreshes the status dashboard PNG
auto_refresh.py — change detection
before = {rid: {"send_time": r.send_time, "active": r.active}
          for rid, r in config_loader.get_all_reports().items()}

config_loader.refresh()
sync_scheduler(scheduler, config_loader)

for rid, r in config_loader.get_all_reports().items():
    if rid not in before:
        changes.append(f"➕ New report: `{rid}` @ {r.send_time}")
    elif before[rid]["send_time"] != r.send_time:
        changes.append(f"🕐 `{rid}` rescheduled: {before[rid]['send_time']} → {r.send_time}")
    elif before[rid]["active"] != r.active:
        flag = "activated ✅" if r.active else "deactivated ❌"
        changes.append(f"🔄 `{rid}` {flag}")
Thread Guard
thread_guard.py
Safety Utility
Prevents starting a second auto-refresh thread when a Jupyter notebook cell is re-run. Checks whether the existing thread is still alive before creating a new daemon thread — a simple but essential guard in a long-running notebook.
thread_guard.py — duplicate thread prevention
if 'refresh_thread' not in globals() or not refresh_thread.is_alive():
    refresh_thread = threading.Thread(
        target=auto_refresh_scheduler,
        args=(scheduler, config_loader, full_config, 5),
        daemon=True     # dies automatically when the notebook shuts down
    )
    refresh_thread.start()

else:
    print("⚠️ Auto-refresh already running — skipped")
Jupyter Widget
jupyter_widget.py
Manual Send UI
An ipywidgets interactive UI embedded in the Jupyter notebook. Lets operators manually trigger any report on demand — bypassing the schedule, SQL readiness checks, and active flag — useful for urgent resends or testing without touching the scheduler.
  • Dropdown auto-populated from live config — always reflects the current report list
  • Calls run_report_job(force=True, honour_active_flag=False) to bypass all guards
  • Output widget captures and displays logs inline below the button in real time
jupyter_widget.py — manual trigger UI
report_dropdown = widgets.Dropdown(
